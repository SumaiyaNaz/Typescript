Project setup (Project configuration)

Make this setup in all folders where you use ts by this we can see ts file file in console so it make easy to catch error in ts file as before setup we see js file in console so how can we fix error in ts file if we can see only js file , so this setup will help in this problem.

Steps

  • In terminal => tsc --int => typescrip compiler initialize => it will create a file having name tsconfig.json in main directory(folder)
  • In json file : (Compiler Options)
    • Uncomment root directory (all ts file in it ) and out (output) directory (all js file in it )
  • Make two folders of same name as in root and out directory and name area :
    • src
    • dist
  • In json file : (Environment Settings)
    • Set module as esnext as now we are using es means js it can set different for all languages
    • Set target (target means version means on which version we want to work) as es2016 , 2022 as mostly we use this.

Note :

Index file is created in main folder and attach js file with it.

Always run file with live server after setup.

ts file will created in src folder

Now write only tsc in terminal beacuse of the setup now we dont need to write file name. and it also generate all file of js in dist folder after cmpiling ts fie into js

tsc --watch => dont need to write tsc again and again as this command runs ts file like live server and its called watch mode

Shortcut of tsc --watch is => tsc -w

To exit watch mode write Ctrl + C in terminal


Extra :

In json file => Other Outputs => "sourceMap": true, => thats y in js all code goes in index.js.map => and in this file (index.js.map) we see "sources":["../src/index.ts"] => thats y we see ts in compiler due to setup

In index.js there is export so we use type module in js link in html file
